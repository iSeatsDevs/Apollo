# Input

Inputs are used for any component that requires a value from the user. Inputs base styles and components are reused
in other component type components such as `<DatePicker/>` and `<NumberInput/>`.

## No Border

To disable the border on the input you can pass the `noBorder` prop 
to the component. This is mainly used when placed on a colored background.

<Preview>
  <Story name="No Border">
    <div style={{ background: '#0174E3', padding: '25px' }}>
      <Input noBorder placeholder="This input does not have a border" />
    </div>
  </Story>
</Preview>

## Validation

Validation can be easily set by using the `danger` or `success` prop. This will change the border color and
text color of a message if applied. We can also use the `message` prop to give additional feedback to the user.

<Preview>
  <Story name="Validation">
    <>
      <div>
        <Input
          defaultValue="Apollo"
          success
          message="Oops! Something went wrong"
        />
      </div>
      <div>
        <Input
          defaultValue="Apollo"
          danger
          message="Oops! Something went wrong"
        />
      </div>
    </>
  </Story>
</Preview>

## Clearable

You can use the `clearable` prop to allow the user to easily clear the input field.

<Preview>
  <Story name="Clearable">
    <Input placeholder="Placeholder" defaultValue="johndoe@gmail.com" clearable />
  </Story>
</Preview>

## Visibility

You can disable the visibility of a field using the `hidden` prop. If you want the user
to be able to toggle between a visible and hidden state you can also enable `visibilityControls` prop.

<Preview>
  <Story name="Visibility">
    <>
      <div>
        <Input placeholder="Password" defaultValue="password" clearable hidden/>
      </div>
      <div>
        <Input placeholder="Password" defaultValue="password" clearable hidden visibilityControls/>
      </div>
    </>
  </Story>
</Preview>
